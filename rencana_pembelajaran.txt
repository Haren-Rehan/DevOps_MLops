### ðŸŽ¯ *FASE 1: FONDASI (Bulan 1-3)*
*Tujuan:* Membangun dasar teknikal yang kuat untuk automation dan deployment model ML.
*Skill Wajib:*
*   *Python & Bash Scripting:* Otomasi tugas dan membangun aplikasi.
*   *Git & GitHub:* Version control dan kolaborasi.
*   *Linux & Terminal:* Mengoperasikan sistem dan server.
*   *Docker Basics:* Membungkus aplikasi dan model ke dalam container.

*Proyek yang Akan Dibuat:*
*"Deploy Model Regresi Sederhana dengan Docker dan REST API"*

2.  *Bungkus dengan Docker:* Tulis Dockerfile untuk membuat image yang berisi model, dependencies, dan aplikasi web sederhana (menggunakan Flask/FastAPI).
3.  *Buat REST API:* API yang menerima input data (misalnya: luas tanah, jumlah kamar) dan mengembalikan prediksi harga dari model.
4.  *Versioning dengan Git:* Kelola semua kode dengan Git dan push ke repository GitHub.

*Contoh Output:* Sebuah image Docker yang bisa dijalankan. Saat dijalankan, Anda bisa mengakses http://localhost:5000/predict untuk mendapatkan prediksi dari model ML.

### ðŸš€ *FASE 2: DEVOPS FOR MLOPS (Bulan 4-6)*
*Tujuan:* Memahami infrastruktur cloud dan pipeline automation untuk deployment yang scalable dan robust.
*Skill Wajib:*
*   *Kubernetes Fundamentals:* Mengelola dan meng-orchestrate container dalam cluster.
*   *CI/CD (GitHub Actions/GitLab CI):* Mengotomasi testing dan deployment.
*   *Terraform Basics:* Mendefinisikan dan menyediakan infrastruktur sebagai kode (IaC).
*   *1 Cloud Provider (AWS/Azure/GCP):* Menjalankan workload di cloud.
*   *Go Programming Basics:* Memahami syntax untuk membantu membaca dan menulis script DevOps.

*Proyek yang Akan Dibuat:*
*"Setup CI/CD Pipeline untuk Deploy Aplikasi ke Kubernetes Cluster di Cloud"*
1.  *Provision Infrastruktur:* Gunakan Terraform untuk membuat Kubernetes cluster (misalnya Amazon EKS atau Google GKE) dan container registry di cloud pilihan.
2.  *Setup CI/CD Pipeline:* Buat workflow GitHub Actions/GitLab CI yang secara otomatis:
    *   Membuild image Docker dari proyek Fase 1.
    *   Mendpush image ke container registry (e.g., AWS ECR, Google Container Registry).
    *   Melakukan rolling update deployment di Kubernetes cluster.
3.  *Deploy ke Kubernetes:* Tulis file manifest Kubernetes (Deployment & Service) untuk menjalankan model Anda di cluster.

*Contoh Output:* Sistem di mana setiap kali Anda push kode baru ke branch main, pipeline CI/CD akan secara otomatis membuild dan mendeploy model terbaru ke Kubernetes cluster di cloud.

### ðŸ¤– *FASE 3: MLOPS CORE (Bulan 7-9)*
*Tujuan:* Mengintegrasikan tool MLOps khusus untuk mengelola siklus hidup model ML secara end-to-end.
*Skill Wajib:*
*   *MLflow:* Melacak eksperimen, mengemas kode, dan mengelola Model Registry.
*   *Apache Airflow:* Mengatur dan memonitor workflow data dan ML yang kompleks.
*   *Model Serving (KServe/Seldon Core):* Melayani model dalam skala production dengan fitur canggih (canary deployment, explainability).
*   *Spark untuk Data Processing:* Memproses data dalam skala besar.

*Proyek yang Akan Dibuat:*
*"Bangun Complete MLOps Pipeline dengan Retraining Otomatis"*
1.  *Integrasi MLflow:* Gunakan MLflow untuk melacak berbagai eksperimen model, log metrik, dan register model terbaik ke Model Registry.
2.  *Bangun Pipeline dengan Airflow:* Buat DAG Airflow yang menjalankan workflow:
    *   *Preprocessing Data:* Gunakan Spark untuk membersihkan dan memproses data baru.
    *   *Training Model:* Script training yang memanfaatkan MLflow untuk melacak run.
    *   *Evaluasi Model:* Bandingkan model baru dengan model production.
    *   *Promosi Model:* Jika model baru lebih baik, promosikan ke staging di MLflow Model Registry.
3.  *Advanced Model Serving:* Deploy model dari registry menggunakan KServe atau Seldon Core di Kubernetes, alih-alih deployment Docker sederhana.

*Contoh Output:* Sebuah sistem yang secara terjadwal (atau terpicu oleh data baru) melakukan retraining, mengevaluasi model, dan siap untuk deployment baru jika model memenuhi syarat.

### ðŸ“Š *FASE 4: PRODUCTION GRADE (Bulan 10-12)*
*Tujuan:* Membangun sistem MLOps yang matang, dapat diandalkan, dan efisien dengan fokus pada data dan monitoring.
*Skill Wajib:*
*   *Feature Store (Feast):* Menyimpan, mengelola, dan melayani feature secara konsisten untuk training dan inference.
*   *Monitoring (Prometheus + Grafana + Evidently AI):* Memonitor kesehatan sistem, performa model, dan drift data.
*   *Data Versioning (DVC):* Melacak versi dataset dan model seperti Git melacak kode.
*   *Advanced Kubernetes:* Network policies, resource management, dan security context.

*Proyek yang Akan Dibuat:*
*"End-to-End Production System dengan Monitoring & Auto-retraining"*
1.  *Implementasi Feature Store:* Integrasikan Feast ke dalam pipeline. Alih-alih membaca langsung dari file, pipeline training dan inference mengambil feature dari Feast.
2.  *Data & Model Versioning:* Gunakan DVC untuk mengversion dataset dan mengelolanya bersama kode di repository Git.
3.  *Setup Comprehensive Monitoring:*
    *   *Prometheus & Grafana:* Untuk memonitori metrik infrastruktur Kubernetes (CPU, memory) dan metrik bisnis aplikasi.
    *   *Evidently AI:* Untuk memonitor drift data dan degradasi performa model pada data inference.
4.  *Bangun Pipeline Auto-retraining:* Tingkatkan pipeline Airflow untuk secara otomatis memicu retraining ketika monitoring mendeteksi model drift atau performa menurun.

*Contoh Output:* Sebuah sistem MLOps production-grade yang memiliki siklus hidup model yang sepenuhnya otomatis, terpantau, dan dapat diandalkan, dari data hingga deployment dan monitoring.
